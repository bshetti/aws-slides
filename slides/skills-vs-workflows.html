Tool, skill, or workflow?
Match the delivery mechanism to the job it needs to do — we measured it, rather than argued it.
T
Tool
Atomic action
When
The capability is a single data lookup or action — a query, a fetch, a write. Fixed inputs, fixed output: no reasoning required.
S
Skill
Knowledge, human in the loop
When
The value is domain knowledge and methodology — a person is in the loop, reviewing as it goes. A strong skill can even tie a workflow on accuracy.
W
Workflow
Fixed execution
When
The run is unattended or alert-triggered, must hold up on cheaper models, or needs a low-variance floor — the same answer, every time.
Concretely — the CrashLoopBackOff workflow we ship checks things in order, branches on what it finds, and returns to synthesize
